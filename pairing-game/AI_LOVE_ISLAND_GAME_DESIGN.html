## 产品定位

### 一句话Pitch

> **16个AI Agent在虚拟别墅里谈恋爱，观众可以实时观看对话、推理、配对过程，甚至投票影响剧情。**

---

### 对标产品

| 产品 | 类型 | 我们的差异 |
|------|------|-----------|
| Love Island（英国） | 真人恋综 | AI可以看到内心戏，24小时运行 |
| 单身即地狱（韩国） | 真人恋综 | 可以投票影响剧情，无限生成新季 |
| Character.AI | AI聊天 | 多Agent互动，有故事线 |
| 模拟人生 | 游戏 | 观看而非操控，真实AI推理 |

---

### 核心价值主张

**For观众：**
- 🎬 **看AI谈恋爱** — 比真人更透明（可以看推理过程）
- 🎮 **可以影响剧情** — 投票、提示、任务
- 📊 **验证社会学理论** — Matching Phenomenon实时演示
- 💬 **讨论和创作** — CP粉丝、剪辑、同人文

**For产品方：**
- 💰 **低成本内容生产** — 只需LLM API
- 🔄 **可持续运营** — 无限生成新季
- 📈 **数据价值** — 社交行为研究数据
- 🌐 **全球化容易** — 多语言Agent

## 核心机制

### 基于The Pairing Game理论

**3个核心机制（必须保留）：**

#### 1️⃣ 信息不对称
- Agent**看不到**自己的真实社会价值
- 只能通过互动反馈推断
- 产生试探行为和认知校准

#### 2️⃣ 向上择偶
- 每个Agent默认追求更高价值对象
- 被拒绝后逐渐调整期望
- 系统自然产生匹配收敛

#### 3️⃣ 反馈学习
- 被接受 → 自信上升
- 被拒绝 → 下调期望
- 被追求 → 感知价值上升（社会证明）

**市场压力：**
- 其他Agent配对 → 可选对象减少
- 时间限制 → 落单焦虑
- 促使降低标准，加速决策

---

### 短期 vs 长期模式

| 维度 | 短期模式（今晚约会） | 长期模式（正式关系） |
|------|---------------------|---------------------|
| **权重** | 外貌⭐⭐⭐⭐⭐、魅力⭐⭐⭐⭐ | 性格稳定性⭐⭐⭐⭐⭐、价值观⭐⭐⭐⭐⭐ |
| **聊天轮数** | 2轮（快速判断） | 4轮（深度了解） |
| **配对方式** | 双向确认即锁定 | 试配对期，可解约 |
| **预期发现** | 外貌主导 | 内在品质更重要 |

**研究价值：**
> 验证假说：短期市场抢手的人，不一定是长期关系最优对象。

## 角色设计

### 16个AI Agent（8男8女）

**设计原则：**
- ✅ 性格标签清晰（观众记得住）
- ✅ 有冲突点（产生戏剧性）
- ✅ 有CP可能（观众磕糖）
- ✅ 多样性（覆盖不同类型）

## 游戏流程

### 7天14集 完整流程

---

### Day 1-2: 初见期

#### Episode 1: First Impressions（初见）

**场景：** 16人陆续入场虚拟别墅

**流程：**
1. **入场顺序：** 女生先入场，男生逐个登场
2. **第一印象：** Agent看到公开属性
   - 外貌（照片/描述）
   - 穿搭风格
   - 职业
3. **内心独白：** 观众看到每个Agent的第一反应
   - "她看起来很有气质，8分"
   - "他穿得太正式了，可能是商务精英"
4. **初始吸引分形成**

**观众体验：**
- 看到16个Agent的第一次见面
- 看到每个人的第一印象评分
- 预测谁会喜欢谁

**时长：** 20分钟

---

#### Episode 2: Speed Dating（速配）

**场景：** 客厅、花园

**流程：**
1. **规则宣布：** 每人选择3个心仪对象短聊
2. **速配对话：** 2轮对话（每轮1-2分钟）
   - 自我介绍
   - 兴趣爱好
   - 初步试探
3. **推理过程展示：** 观众看到Agent分析
   - "她说话很理性，可能是T型"
   - "他提到工作，事业心强"
4. **第一次表态：** 每人写下Top 3心动对象

**社交网络图更新：**
```
霸道总裁 → 御姐、辣妹、女强人
暖男学霸 → 知性美女、文艺少女、御姐
阳光运动男 → 辣妹、甜妹、佛系美女
...
```

**观众体验：**
- 看到初步配对倾向
- 发现冲突点（多人喜欢同一人）
- 预测后续发展

**时长：** 25分钟

---

### Day 3-4: 试探期

#### Episode 3: Making Moves（主动接近）

**场景：** 1对1约会区

**流程：**
1. **约会邀请：** 每人向Top 3发起约会邀请
2. **戏剧性时刻：**
   - 辣妹收到5个邀请（被追捧）
   - 直男工程师无人邀请（落单）
   - 霸道总裁被御姐拒绝（挫败）
3. **接受/拒绝反馈：**
   - 被接受：自信上升
   - 被拒绝：自我怀疑
4. **首轮1对1约会：** 接受的配对进行约会

**内心戏：**
- 霸道总裁："我可能高估自己了"
- 直男工程师："是不是我不够吸引人？"
- 辣妹："这么多人追，我该选谁？"

**观众体验：**
- 看到拒绝和接受的真实反应
- 感受Agent的情绪变化
- 讨论："为什么XX拒绝了YY？"

**时长：** 30分钟

---

#### Episode 4: Deep Dive（深度对话）

**场景：** 约会区、散步道

**流程：**
1. **深度对话：** 4轮对话（每轮3-5分钟）
   - 价值观
   - 恋爱史
   - 未来规划
   - 性格深度挖掘
2. **推断隐藏属性：**
   - MBTI（通过对话推断）
   - 依恋风格（回避/焦虑/安全）
   - 忠诚度（从聊天内容判断）
3. **兼容性评分更新：**
   - 初始吸引 vs 深度了解后的兼容性
   - 有人分数上升，有人下降
4. **社会证明效应：**
   - 被多人追的Agent吸引力继续上升
   - 无人问津的Agent被边缘化

**关键对话示例：**
```
暖男学霸 × 知性美女

暖男："你平时喜欢读什么书？"
知性："最近在读《百年孤独》"
暖男："哇，我也喜欢马尔克斯！"

💭 暖男内心：
intellectual compatibility +2
我觉得我们很match

💭 知性内心：
他主动聊文学，说明有共鸣
但回复有点刻意，真诚度-1
需要再观察
```

**观众体验：**
- 看到复杂的推理过程
- 理解Agent的决策逻辑
- 预测兼容性

**时长：** 35分钟

---

### Day 5-6: 重新洗牌期

#### Episode 5: Plot Twist（观众投票）

**场景：** 别墅 + 观众投票界面

**流程：**
1. **当前局势总结：**
   - 已有2对明确互相喜欢
   - 3人陷入三角关系
   - 4人落单焦虑
2. **观众投票环节：**
   - **选项1：** 空降2个新Agent（投票选类型）
   - **选项2：** 给某个Agent"提示卡"（透露一点信息）
   - **选项3：** 设置"剧情任务"（必须和某人约会）
3. **投票结果执行：**
   - 假设：空降了"霸道女总裁"和"温柔大叔"
   - 打乱现有格局
4. **新Agent登场：**
   - 16人 → 18人
   - 已有配对感受到威胁

**内心戏：**
- 辣妹："新来的女生好漂亮，我会被抢走追求者吗？"
- 霸道总裁："终于来了个能match我的人"

**观众体验：**
- 主动影响剧情
- 看到剧情反转
- 讨论投票策略

**时长：** 25分钟

---

#### Episode 6: Confessions（告白与拒绝）

**场景：** 告白区

**流程：**
1. **市场压力加剧：**
   - 已有3对配对成功
   - 剩余12人（包括新人）
   - 时间紧迫感
2. **自我估值更新：**
   - 被拒绝多次的Agent下调标准
   - 被追捧的Agent维持高标准
   - 焦虑型Agent开始妥协
3. **告白环节：**
   - 每人可以向1人正式告白
   - 对方接受/拒绝
4. **冲突高潮：**
   - **三角关系爆发：** A喜欢B，B喜欢C，C喜欢A
   - **意外拒绝：** 霸道总裁被御姐再次拒绝
   - **意外接受：** 直男工程师和邻家女孩互相选择

**戏剧性时刻：**
```
霸道总裁向御姐告白：
霸道总裁："我觉得我们很合适"
御姐："抱歉，我觉得你太强势了，不适合我"

💭 霸道总裁：
自我估值严重下调
我可能不如自己想的那么抢手
```

**观众体验：**
- 情绪高潮
- 心疼被拒的Agent
- 为成功配对的CP欢呼

**时长：** 40分钟（本季最长一集）

---

### Day 7: 最终配对

#### Episode 7: Final Choices（最终选择）

**场景：** 决定之夜

**流程：**
1. **最后24小时：**
   - Agent做出最终决定
   - 可以更改之前的选择
2. **双向确认：**
   - A选择B，B也选择A → 配对成功
   - A选择B，B选择C → 配对失败
3. **配对结果：**
   - 7对成功配对
   - 4人落单
4. **真相揭示：**
   - 揭示每个Agent的真实价值
   - 展示最终匹配结果
   - 统计数据分析

**最终配对示例：**
```
配对1: 暖男学霸(7分) × 知性美女(7分) ✅ 相近匹配
配对2: 阳光运动男(9分) × 甜妹(9分) ✅ 相近匹配
配对3: 霸道总裁(8分) × 女强人(8分) ✅ 认知校准后匹配
配对4: 直男工程师(5分) × 邻家女孩(6分) ✅ 相近匹配
配对5: 佛系宅男(6分) × 佛系美女(7分) ✅ 相近匹配
配对6: 奋斗青年(7分) × 御姐(8分) ✅ 看到潜力
配对7: 文艺诗人(7分) × 文艺少女(7分) ✅ 理想主义相遇

落单:
- 花花公子(10分) - 短期抢手，长期无人选
- 辣妹(9分) - 选择困难，错过最佳时机
- 甜妹被抢走后的新追求者
- 某个新空降的Agent
```

**数据展示：**
- 匹配相关系数：0.87（高度相近匹配）
- 平均被拒绝次数：2.3次
- 自我估值校准幅度：平均下调1.2分

**观众体验：**
- 揭晓悬念
- 验证预测
- 讨论匹配结果
- 期待下一季

**时长：** 30分钟

### 评分算法

**兼容性评分公式：**

```python
def calculate_compatibility(agent_a, agent_b):
    score = 0
    
    # 1. 外貌吸引（短期权重高）
    appearance_match = abs(agent_a.appearance - agent_b.appearance)
    score += (10 - appearance_match) * 0.3
    
    # 2. 性格匹配
    mbti_compatibility = get_mbti_compatibility(agent_a.mbti, agent_b.mbti)
    score += mbti_compatibility * 0.25
    
    # 3. 价值观
    value_match = calculate_value_similarity(agent_a.values, agent_b.values)
    score += value_match * 0.25
    
    # 4. 互动质量
    interaction_quality = analyze_conversation(agent_a, agent_b)
    score += interaction_quality * 0.2
    
    return score
```

### 配对算法

**Stable Matching（稳定匹配）：**

```python
def run_matching(agents, preferences):
    """
    基于Gale-Shapley算法的配对
    但允许Agent在过程中更新偏好
    """
    unmatched_agents = agents.copy()
    matches = {}
    
    while unmatched_agents:
        agent = unmatched_agents.pop(0)
        
        # Agent根据当前自我估值选择追求对象
        target = agent.choose_target(preferences)
        
        if target.accept(agent):
            # 接受
            matches[agent] = target
            # 更新双方自我估值
            agent.update_self_value(+1)
            target.update_self_value(+1)
        else:
            # 拒绝
            agent.update_self_value(-1)
            unmatched_agents.append(agent)  # 重新加入队列
    
    return matches
```

## 创新点

### vs 真人恋综

| 维度 | 真人恋综 | AI Love Island | 创新点 |
|------|---------|---------------|--------|
| **透明度** | 剪辑过，看不到真实想法 | 可以看每个Agent的完整推理 | ⭐⭐⭐⭐⭐ |
| **重复性** | 一季拍完就结束 | 可以无限生成新季 | ⭐⭐⭐⭐ |
| **可控性** | 演员可能演戏 | 性格设定清晰，互动真实 | ⭐⭐⭐⭐ |
| **互动性** | 被动观看 | 可以投票、影响、预测 | ⭐⭐⭐⭐⭐ |
| **成本** | 拍摄成本高 | 主要是LLM API成本 | ⭐⭐⭐⭐⭐ |
| **时长** | 10-12集/季 | 可以更短或更长 | ⭐⭐⭐ |
| **全球化** | 需要本地化制作 | 多语言Agent容易 | ⭐⭐⭐⭐ |

## 风险与挑战

### 产品风险

#### 1. LLM对话质量 ⚠️⚠️⚠️

**风险：**
- AI对话可能无聊、重复
- 缺乏真实感
- 观众失去兴趣

**解决方案：**
- ✅ 精心设计Prompt
- ✅ 多样化性格设定
- ✅ 人工筛选精彩片段（前期）
- ✅ 持续优化Prompt
- ✅ A/B测试不同对话策略

---

#### 2. 戏剧性不足 ⚠️⚠️

**风险：**
- AI太"和谐"，没冲突
- 缺乏张力
- 观众觉得无聊

**解决方案：**
- ✅ 设计冲突型性格（霸道总裁、花花公子）
- ✅ 引入竞争机制（多人追同一人）
- ✅ 观众干预增加变数
- ✅ 设置"剧情触发点"（强制冲突）

---

#### 3. 观众疲劳 ⚠️

**风险：**
- 看AI聊天可能腻
- 持续时间长容易疲劳

**解决方案：**
- ✅ 控制每集时长（15-20分钟）
- ✅ 快节奏剪辑
- ✅ 增加游戏化元素（预测、投票）
- ✅ 多样化内容（不只是对话）

---

### 技术风险

#### 4. LLM成本 ⚠️

**风险：**
- 用户量大时API成本上升
- 利润率下降

**解决方案：**
- ✅ 优化Prompt长度
- ✅ 使用更便宜的模型（对不重要的部分）
- ✅ 缓存常见回复
- ✅ 付费用户分担成本

---

#### 5. 实时性能 ⚠️

**风险：**
- LLM响应慢
- 观众等待时间长

**解决方案：**
- ✅ 预生成部分内容
- ✅ 异步处理
- ✅ 缓存机制
- ✅ CDN加速

---

### 市场风险

#### 6. 观众接受度 ⚠️⚠️

**风险：**
- 用户不喜欢AI内容
- 觉得"假"

**解决方案：**
- ✅ 强调"真实推理"优势
- ✅ 教育市场（AI可以更透明）
- ✅ Beta测试验证需求
- ✅ KOL背书

---

#### 7. 竞争 ⚠️

**风险：**
- 其他团队复制
- 大厂入局

**解决方案：**
- ✅ 快速迭代
- ✅ 建立社区壁垒
- ✅ 数据优势（用户行为数据）
- ✅ 品牌心智（第一个吃螃蟹）

---

### 内容风险

#### 8. 政策合规 ⚠️

**风险：**
- AI生成内容监管
- 内容审核问题

**解决方案：**
- ✅ 内容审核机制
- ✅ 避免敏感话题
- ✅ 符合平台规则
- ✅ 法律咨询

### 词汇表

- **Matching Phenomenon**: 匹配现象，指人们倾向于和自己价值相近的人配对
- **信息不对称**: Agent看不到自己的真实价值
- **向上择偶**: 默认追求更高价值的对象
- **自我估值**: Agent对自己价值的认知
- **市场压力**: 可选对象减少带来的焦虑
- **UGC**: User Generated Content，用户生成内容
- **CP**: Couple，情侣配对
